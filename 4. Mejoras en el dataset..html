Como se puede observar, la distribución del tipo de producto es diferente si es un cliente o no lo es. Los usuarios que acaban comprando consumen un mayor número de fichas básicas que de perfiles promocionales. Si se recuerda, la oferta de los perfiles promocionales únicamente es durante el primer mes después de registrarse, además, únicamente se pueden consumir hasta cinco perfiles promocionales, mientras que se pueden consumir todas las fichas básicas que se quieran. Esta distribución puede estar diciendo que los usuarios que acabaran comprando, al estar interesados en los producos que oferta el ecommerce, consumen más productos de tipo ficha básica al explorar el site, mientras que los usuarios que no compran, al consumir menos productos, tienen un mayor porcentaje de perfiles promocionales consumidos.

Se va a hacer una pequeña agrupación a modo de analizar el número de consumos de perfiles promocionales dependiendo de si el usuario es un cliente o no, para explorar si es necesario ofertar 5 perfiles promocionales durante el mes de la oferta o realmente no se estan utilizando para los usuarios que acaban transformándose en clientes.

Manipulación y consolidación ¶

Debido a esta nueva exploración, se va a hacer la siguiente agregación de estas columnas. Como se tiene un dataframe con todos los consumos por usuarios y un único usuario puede consumir más de un tipo de producto de diferentes empresas que esten en distintos departamentos, se utilizará la moda de las tres nuevas variables. De esta forma para cada usuario se obtendrá la información más recurrente de cada variable. Otra posibilidad podría haber sido el obtener el número de consumos de perfil promocional y el de ficha básica por separado, pero como el perfil promocional sólo se puede consumir como máximo 5 veces, el valor de número de consumos totales sería muy similar a los de ficha básica, por lo que tiene más sentido tener el total y el valor modal del tipo, recogiendo la información de forma similar.

from sklearn.preprocessing import OneHotEncoder

def oneHotEncoding(dataRaw,columnName):
    """Función que transforma una variable categórica con one-hot-encoding, elimina la original
       y cambia el nombre de las nuevas columnas a un formato descriptivo """
    
    OHencoder = OneHotEncoder(sparse_output=False)
    # se crean nuevas columnas binarias
    columnDataEncoded = pd.DataFrame(OHencoder.fit_transform(dataRaw[[columnName]]))
    # se cambia el nombre de las nuevas columnas para que sea más descriptivo
    columnDataEncoded.columns = OHencoder.get_feature_names_out([columnName])
    # se añaden las nuevas columnas al dataframe original menos la columna transformada
    dataTransformed = pd.concat([dataRaw.drop([columnName], axis=1), columnDataEncoded], axis=1)
    return dataTransformed

from sklearn.feature_selection import chi2, SelectKBest, f_classif

def applyTest(data, columns, target, test, mode):
    """Esta función utiliza la función SelectKBest de sklearn para obtener los estadísticos dependiendo
       del test que se quiera aplicar y visualizarlos. En caso de querer obtener los mejores atributos
       se puede utilizar la variable 'mode' que para se devuelvan los top K atributos."""
    
    X = data[columns]
    if test == chi2:
        Y = data[[target]]
    if test == f_classif:
        Y = data[[target]].values.ravel()
        
    computeTest = SelectKBest(test, k = mode)
    results = computeTest.fit(X,Y)

    # se visualiza el p_value y el estadistico que devuelve la función dependiendo del test
    fig, axs = plt.subplots(1,2,figsize=(12,3))
    sns.barplot(x=results.feature_names_in_,y=results.pvalues_, color="steelblue", ax=axs[0]).set_title("pvalue")
    if test == chi2:
        
        axs[0].axhline(0.05,color='cornflowerblue',linestyle='dashed')

    sns.barplot(x=results.feature_names_in_,y=results.scores_, color="steelblue", ax=axs[1]).set_title("score")

    for ax in fig.axes:
        plt.sca(ax)
        plt.xticks(rotation=45, horizontalalignment="right")

    plt.show()

    return None
